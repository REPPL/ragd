# Architecture Decisions

Documentation of significant technical decisions.

## Structure

| Directory | Purpose |
|-----------|---------|
| [adrs/](./adrs/) | Architecture Decision Records |

## When to Create an ADR

Create an ADR when:
- Making a significant architectural choice
- Choosing between competing technologies
- Establishing a pattern that affects multiple components
- Making a decision that's hard to reverse

## What Belongs Here

- Architecture Decision Records (ADRs)
- Technology choice documentation
- Design rationale

## What Doesn't Belong Here

- Implementation details → [implementation/](../implementation/)
- Feature plans → [features/](../features/)
- Development narrative → [process/devlogs/](../process/devlogs/)
