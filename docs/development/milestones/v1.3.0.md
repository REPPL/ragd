## Focus Areas

### REST API (F-133)

```bash
# Start API server
ragd serve --port 8080

# Endpoints
POST /api/v1/index          # Index documents
GET  /api/v1/search         # Search
POST /api/v1/chat           # Chat
GET  /api/v1/status         # Status
GET  /api/v1/documents      # List documents
```

### OpenAPI Documentation

- Auto-generated from endpoints
- Interactive Swagger UI
- Client SDK generation

### API Authentication (F-135)

```bash
# Generate API key
ragd api-key create --name "my-app"

# Use in requests
curl -H "Authorization: Bearer <key>" http://localhost:8080/api/v1/search
```

**Features:**
- API key generation and management
- Rate limiting per key
- Scope-based permissions
- Key rotation support

### Webhooks (F-136)

```bash
# Register webhook
ragd webhook create --event document.indexed --url https://example.com/hook

# Events
- document.indexed
- document.deleted
- search.performed
- chat.completed
```

### Health Endpoints (F-137)

```bash
# Kubernetes-ready health checks
GET /health      # Always returns 200 if running
GET /ready       # Returns 200 when ready to serve
GET /metrics     # Prometheus-compatible metrics
```
