## Implementation Notes

### Pragmatic Decisions

1. **SQLite over Kuzu**: Knowledge graph uses SQLite storage instead of Kuzu. Simpler, no new dependencies, sufficient for local use.

2. **Pattern over ML extraction**: Entity extraction uses regex patterns by default. spaCy is optional for higher quality.

3. **Deferred features**: Dual-index architecture, temporal queries, and persona system deferred to future releases.

## CLI Commands Added

```bash
# Tier management
ragd tier set <doc_id> <tier>
ragd tier show <doc_id>
ragd tier list [tier]
ragd tier summary

# Tag suggestions
ragd tag suggestions
ragd tag confirm <doc_id> <tag>

# Collections
ragd collection create <name> --include-all "tag1,tag2"
ragd collection search <name> <query>

# Tag library
ragd tag library create <namespace>
ragd tag library add <namespace> <tag>
```
