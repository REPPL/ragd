## Implementation Notes

### Pragmatic Decisions

1. **SQLite over Kuzu**: Knowledge graph uses SQLite storage instead of Kuzu. Simpler, no new dependencies, sufficient for local use.

2. **Pattern over ML extraction**: Entity extraction uses regex patterns by default. spaCy is optional for higher quality.

3. **Foundation release**: CLI commands for graph exploration deferred to v1.1.

## Technical Approach

### Storage

SQLite-backed graph storage:
- Entities table (id, name, type, metadata)
- Relationships table (source_id, target_id, type, weight)
- Co-occurrence tracking for relationship strength

### Entity Extraction

Two modes:
- **Pattern-based** (default): Fast regex patterns for common entity types
- **spaCy** (optional): Higher quality NER with `en_core_web_sm`
