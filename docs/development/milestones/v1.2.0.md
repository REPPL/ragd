## Focus Areas

### Onboarding Flow (F-134)

Interactive setup wizard for new users:

```bash
ragd init
# Welcome to ragd!
#
# Detected: Apple M2, 16GB RAM
# Recommended model: llama3.2:3b
#
# ? Install recommended model? [Y/n]
# ? Index a sample document? [Y/n]
# ? Run a test search? [Y/n]
#
# Setup complete! Try: ragd search "hello"
```

**Key Features:**
- Hardware detection and model recommendations
- Optional sample document indexing
- Interactive test to verify installation
- Clear next steps for users

### Target Users

- First-time ragd users
- Non-technical users
- Users migrating from other RAG tools

### Advanced CLI Features

#### Comparison Framework (F-140)

A/B testing for models and configurations:

```bash
# Compare models on same query set
for model in llama3 mistral gemma; do
  ragd search "complex query" --model $model --json >> results.jsonl
done

# Compare backends
ragd search "test query" --backend chroma --json > chroma.json
ragd search "test query" --backend faiss --json > faiss.json
```

#### Batch Operations (F-141)

Process multiple queries or documents efficiently:

```bash
# Batch evaluation with timing
ragd batch-search queries.txt --timing --output results.csv

# Batch indexing with progress
ragd batch-add documents/ --progress
```

#### Results Export (F-142)

Export search results to various formats:

```bash
ragd search "query" --output results.json --format json
ragd search "query" --output results.csv --format csv
```

#### Timing & Metrics (F-143)

Performance metrics for all operations:

```bash
ragd search "query" --timing
# Search completed in 0.342s
# - Embedding: 0.120s
# - Vector search: 0.180s
# - Reranking: 0.042s
```
