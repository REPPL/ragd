# Milestone: v0.9.6 - Alpha Testing Release

Final polish for first external alpha testers.

## Theme

**"Ready for Alpha"** - Feature-complete core with comprehensive feedback loops.

## Priorities for Alpha Testing

### High Priority (Must Have)

1. **Error Logging & Recovery (F-111)**
   - Per-document success/failure tracking
   - Categorised failure reasons
   - Remediation hints for common errors

2. **CLI User Feedback (F-114)**
   - Clean progress indicators
   - Descriptive error messages
   - Actionable hints with errors

3. **Dry-Run Mode (F-118)**
   - `--dry-run` for destructive operations
   - Preview changes before committing

### Medium Priority (Should Have)

4. **Operation Audit Trail (F-112)**
   - Log all operations with timestamps
   - Queryable operation history

5. **Source Quality Scoring (F-115)**
   - Confidence scores for extracted content
   - Quality indicators in search results

### From Deferred Features

- F-098: Advanced HTML Engine (JavaScript rendering)
- F-099: PDF Layout Intelligence
- F-105: Indexing Self-Evaluation

## Alpha Testing Focus Areas

### Stability Testing
- Large document collections (1000+ files)
- Mixed file types
- Error recovery scenarios
- Interrupted operations

### Usability Testing
- First-time setup experience
- Configuration wizard
- Error message clarity
- Documentation completeness

### Performance Testing
- Indexing speed
- Search latency
- Memory usage
- Storage efficiency

## Success Criteria

- [ ] Zero crashes on malformed input
- [ ] Clear error messages for all failures
- [ ] Documentation covers all features
- [ ] First-time users can complete setup in < 10 minutes
- [ ] Search returns relevant results within 500ms

## Dependencies

- v0.9.5 (Stability & Logging)
