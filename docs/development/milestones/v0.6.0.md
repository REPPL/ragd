# Milestone: v0.6.0 - Storage

Flexible storage backends and the LEANN integration experiment.

## Goal

Abstract the storage layer to support multiple vector database backends. This enables users to choose the best storage solution for their needs and opens the door to the exciting LEANN (Learned Approximate Nearest Neighbour) experiment.

## Theme

**"Your Data, Your Way"** - Storage flexibility without lock-in.

## Draft Release Notes

### ragd v0.6.0 - Storage

**Your data, your way.**

Choose the vector store that fits your needs. Or experiment with the bleeding edge.

**New Features:**

- **Storage Abstraction** - Swap backends without changing your workflow
- **FAISS Support** - Facebook's battle-tested similarity search
- **LEANN Experiment** - Custom learned index for research
- **Migration Tools** - Move data between backends

**Getting Started:**

```bash
# Default remains ChromaDB
ragd index ~/Documents/

# Use FAISS backend
ragd config set storage.backend faiss
ragd index ~/Documents/

# Migrate existing data
ragd migrate --from chromadb --to faiss
```

**What's Next:**

v0.7 focuses on privacy: PII detection and encryption for sensitive documents.
