# Milestone: v0.6.0 - Storage

Flexible storage backends with protocol-based abstraction.

## Goal

Abstract the storage layer to support multiple vector database backends. This enables users to choose the best storage solution for their needs.

## Theme

**"Your Data, Your Way"** - Storage flexibility without lock-in.

## Release Notes

### ragd v0.6.0 - Storage

**Your data, your way.**

Choose the vector store that fits your needs with protocol-based backend abstraction.

**New Features:**

- **Storage Abstraction** - VectorStore protocol for pluggable backends
- **FAISS Support** - High-performance similarity search with auto-index selection
- **Score Normalisation** - Consistent 0-1 scores across all backends (critical for Agentic RAG)
- **SQLite Metadata Proxy** - ChromaDB-style filtering for FAISS
- **Backend CLI** - `ragd backend show/list/set/health/benchmark`
- **Model Cards** - Hardware-aware model recommendations
- **Performance Profiler** - Benchmark storage operations

**Getting Started:**

```bash
# Default remains ChromaDB
ragd index ~/Documents/

# Check available backends
ragd backend list

# Benchmark your backend
ragd backend benchmark

# Get model recommendations
ragd models recommend --use-case research
```

**What's Next:**

v0.7 focuses on privacy: PII detection and encryption for sensitive documents.
