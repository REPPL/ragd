# v0.9.5 Implementation Record

**Theme:** "Ready for Early Adopters" - Stability & Logging
**Released:** 2025-12-05

## Overview

v0.9.5 delivers stability and observability features for early adopter release: structured logging, consistent exit codes, health checks, and self-healing capabilities.

## Features Implemented

### F-110: Structured Logging

JSON-formatted logging with configurable levels.

**Changes:**
- Created `src/ragd/logging/structured.py` with StructuredLogger
- JSON-formatted log entries
- Configurable console and file levels
- Third-party log suppression (transformers, paddleocr, etc.)
- Global logger configuration

### F-113: Exit Codes & Status

Consistent exit codes for scripting integration.

**Changes:**
- Created `src/ragd/ui/cli/exit_codes.py` with ExitCode enum
- Standard Unix-style exit codes:
  - SUCCESS (0), GENERAL_ERROR (1), USAGE_ERROR (2)
  - CONFIG_ERROR (3), DEPENDENCY_ERROR (4), PARTIAL_SUCCESS (5)
  - NOT_FOUND (6), PERMISSION_ERROR (7), TIMEOUT_ERROR (8)
  - INTERRUPTED (130)
- Description and exception mapping functions

### F-116: Index Integrity Checks

Health check framework via `ragd doctor`.

**Changes:**
- Created `src/ragd/ui/cli/doctor.py` with health check framework
- Database existence check
- Vector store verification
- Document-chunk integrity check
- Configuration file check
- Rich table report output

### F-117: Self-Healing Index

Auto-repair for common issues.

**Changes:**
- Orphaned chunk removal
- Missing directory creation
- Auto-fix with `--fix` flag
- Issue detection and remediation
