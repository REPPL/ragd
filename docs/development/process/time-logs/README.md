# Time Logs

Time tracking for features and versions.

## Purpose

Time logs record actual hours spent on development work. This helps with estimation and planning for future work.

## Available Logs

*No time logs yet - project initialisation in progress.*

## Time Log Template

```markdown
# Time Log: YYYY-MM-DD

## Session
- **Start:** HH:MM
- **End:** HH:MM
- **Duration:** X hours

## Work Done
- Task 1 (X hours)
- Task 2 (X hours)

## Notes
Any relevant observations.
```

## Naming Convention

Files named: `YYYY-MM-DD.md` (e.g., `2025-11-25.md`)

## What Belongs Here

- Time tracking records
- Duration documentation
- Effort estimates vs actuals

## What Doesn't Belong Here

- Development narrative → [devlogs/](../devlogs/)
- Technical details → [implementation/](../../implementation/)
