# Development Logs

Daily development narrative and session logs.

## Purpose

Devlogs capture the story of development - challenges faced, decisions made, and lessons learned. They're written in narrative form for future reference.

## Available Logs

*No devlogs yet - project initialisation in progress.*

## Devlog Template

```markdown
# Devlog: YYYY-MM-DD

## Summary
Brief overview of today's work.

## Progress
What was accomplished.

## Challenges
Problems encountered and how they were resolved.

## Next Steps
What comes next.
```

## Naming Convention

Files named: `YYYY-MM-DD.md` (e.g., `2025-11-25.md`)

## What Belongs Here

- Development narrative
- Challenge documentation
- Session summaries
- Learning notes

## What Doesn't Belong Here

- Time tracking → [time-logs/](../time-logs/)
- Technical details → [implementation/](../../implementation/)
- Architecture decisions → [decisions/adrs/](../../decisions/adrs/)
