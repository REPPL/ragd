# Reference

Technical reference documentation for ragd v0.1.0.

## Purpose

Reference documentation provides accurate, complete technical information. It describes how things work and is information-oriented.

|----------|-------------|
| [Data Schema](./data-schema.md) | Pydantic models, ChromaDB structure, citation metadata |

## Reference Guidelines

When writing reference docs:
- Be accurate and complete
- Use consistent structure
- Include all parameters, options, and return values
- Keep descriptions factual, not tutorial-like

## What Doesn't Belong Here

- Learning materials → [Tutorials](../tutorials/)
- How-to guides → [Guides](../guides/)
- Conceptual explanations → [Explanation](../explanation/)
